# 产品需求文档

## 产品定位

将微信群消息整理工具从"个人脚本"升级为"通用产品"。

目标用户是使用 Claude Code 的人，不一定懂技术。用户通过与 AI 对话完成所有操作，不需要手动运行命令或编辑配置文件。

唯一前提：已安装并登录 Claude Code。

---

## 核心概念

### 三个维度

| 维度 | 说明 |
|------|------|
| 采集 | 从哪些微信群采集消息 |
| 策略 | 用什么视角提取有价值的内容 |
| 输出 | 整理结果输出到哪里 |

### Insight（洞察）

数据单元不只是"问答"，而是群里发生的任何有价值的内容：

| 类型 | 说明 |
|------|------|
| `qa` | 问题与解答 |
| `topic` | 热点话题讨论 |
| `link` | 文章链接与观点 |
| `decision` | 决策与结论 |
| `announcement` | 公告与通知 |

同一份原始消息，不同策略提取出不同类型的 insights。

### 策略模板

内置 3 个模板，用户给群打标签时选择：

| 模板 | 提取重点 |
|------|---------|
| 技术问答群 | 问题、报错、解决方案、工具使用 |
| 资讯分享群 | 文章链接、观点、热点话题 |
| 通用群 | 所有有价值的内容，不限类型 |

用户可在模板基础上追加自定义描述，无需从头写。

策略与群是多对多关系：同一个群可以有多个策略（不同视角），同一个策略可以覆盖多个群。

## 功能需求

### 1. Skill（AI 对话入口）

用户通过 Claude Code 调用 skill，AI 引导完成所有操作。

**首次配置流程：**
1. AI 检测缺少的依赖，自动安装（wx-cli、lark-cli、Python 包）
2. AI 列出所有微信群，用户选择要采集的群
3. 用户为每个群选择策略模板，可追加自定义描述
4. 用户提供飞书文档 URL（输出目标）
5. 配置写入本地数据库，无需手动编辑文件

**日常使用：**
- "采集今天的消息" → AI 运行采集和提取
- "打开看板" → AI 启动本地服务，浏览器自动打开
- "搜索关于 xxx 的内容" → AI 查询数据库，返回结果
- "把这几条导出到飞书" → AI 触发导出

**错误处理：**
- 所有错误信息用中文呈现，不暴露技术细节

### 2. 提取引擎

用 LLM 替换现有的规则驱动提取：

- 输入：原始消息 JSON + 策略描述（模板 + 自定义补充）
- 输出：结构化 insights，写入本地 SQLite
- 保留机械过滤参数：最短字符数、时间窗口（可配置，有默认值）

### 3. 本地数据库（SQLite）

存储所有 insights，支持：
- 按群、日期、类型、标签过滤
- 全文检索（标题 + 内容）
- 记录是否已导出飞书

### 4. 可视化看板（Web UI）

本地 Web 服务，由 skill 自动启动，浏览器自动打开，用户无需知道"服务器"概念。

功能：
- 按群、日期、类型浏览 insights
- 全文搜索 + 字段过滤（标签、热度、类型、导出状态）
- 勾选条目 → 点"导出到飞书"按钮
- 策略管理：查看、新建、编辑策略

语义搜索：先 SQLite 拉候选集，再由 LLM 判断相关性（不引入向量库）。

### 5. 导出飞书

- 从看板勾选条目后触发
- 或通过 skill 对话触发
- 调用现有 `write-feishu-doc.sh`，格式与现有文档兼容

---

## 技术约束

- 只支持 macOS（wx-cli 限制）
- 只支持 Claude Code 作为 AI 入口
- 本地运行，不上传数据到任何服务器
- Python 生态（FastAPI + SQLite），与现有脚本一致
- 前端：Vue 3（CDN 引入），不需要 Node.js 构建
